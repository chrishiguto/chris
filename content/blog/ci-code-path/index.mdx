# Code rides a deploy

<Callout kind="note" title="Edited again, still no code">
  This post was retitled once to exercise the Workers Cache path, and edited
  again here with nothing but prose. A content-only publish flips the `current`
  pointer and purges the `post:ci-code-path` tag through the site worker's
  `/__purge` — no wasm redeploy, no CI build.
</Callout>

The stages widget below is not part of the site's shared vocabulary — it
lives in `components.rs` right next to this file (ADR-0004). Pushing that
component *with* the post is a mixed commit: the pipeline worker parks the
post as pending, fires the publish workflow, and CI builds, checks the size
budget, deploys both workers, and only then drains the post through
`/publish`.

<DeployStages total={4} />

Advance through the stages — build, size gate, deploy, publish. The island
hydrates because the deploy that carried its code always lands first.

Contrast that with an edit like this one: the component already shipped, so
the same file changes flow through the *instant* path instead. Nothing
compiles; only the snapshot and the pointer move. The shared counter below is
proof — it was already in the vocabulary, so dropping it in cost zero builds:
