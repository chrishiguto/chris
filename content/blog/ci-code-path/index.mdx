# Code rides a deploy

<Callout kind="note" title="Retitled again, still no code">
  This post has been retitled a few times now to exercise the Workers Cache
  path, and this pass is more of the same: pure prose. A content-only publish
  flips the `current` pointer and purges the `post:ci-code-path` tag through the
  site worker's `/__purge` — no wasm redeploy, no CI build.
</Callout>

A push can take one of two paths, and which one depends entirely on whether it
carries code. This file has walked both.

The stages widget below is not part of the site's shared vocabulary — it lives
in `components.rs` right next to this file (ADR-0004). Pushing that component
*with* the post was a mixed commit: the pipeline worker parked the post as
pending, fired the publish workflow, and CI built, checked the size budget,
deployed both workers, and only then drained the post through `/publish`.

Advance through the stages — build, size gate, deploy, publish. The island
hydrates because the deploy that carried its code always lands first.

<Callout kind="tip" title="The instant path">
  Contrast that with an edit like this one. The component already shipped, so
  the same file now flows through the *instant* path instead: nothing compiles,
  and only the snapshot and the pointer move.
</Callout>
