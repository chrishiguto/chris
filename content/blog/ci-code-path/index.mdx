# Code rides a deploy

<Callout kind="note" title="Retitled yet again, still no code">
  Another pass, another title, and once more nothing but prose behind it. A
  content-only publish flips the `current` pointer and purges the
  `post:ci-code-path` tag through the site worker's `/__purge` — no wasm
  redeploy, no CI build, no wondering whether the island survived.
</Callout>

Every push lands on one of two roads, and the fork is decided by a single
question: does the commit carry code? This file has travelled both, more than
once.

Take the road with code first. The stages widget below is not part of the
site's shared vocabulary — it lives in `components.rs` right next to this file
(ADR-0004). Shipping that component *with* the post was a mixed commit: the
pipeline worker parked the post as pending, fired the publish workflow, and CI
built, checked the size budget, deployed both workers, and only then drained
the post through `/publish`.

<DeployStages total={4} />

Advance through the stages — build, size gate, deploy, publish. The island
hydrates because the deploy that carried its code always lands before the
content that needs it.

<Callout kind="tip" title="Now the instant road">
  An edit like this one takes the other fork. The component already shipped, so
  the same file skips CI entirely: nothing compiles, nothing redeploys, and only
  the snapshot and the pointer move.
</Callout>
